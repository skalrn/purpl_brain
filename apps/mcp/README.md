# purpl-brain MCP server

Exposes the purpl-brain API to Claude Code as four tools so decisions persist across agent sessions with full attribution.

The point: an agent can recover the *why* behind library choices, rejected approaches, and unresolved questions across sessions — without the developer pasting context.

## Prerequisites

- A running purpl-brain stack (API on `:3001`, Redis, Qdrant, Neo4j, and the three pipeline workers). See repo root for stack setup.
- An API key. Generated by `setup.sh` and written to `apps/mcp/.env`.

## Install

```bash
cd apps/mcp
npm install
npm run build
```

This produces `dist/index.js`. Clients spawn it directly with `node`.

## Claude Code setup

Merge the `mcpServers` block from [`claude-code-config.example.json`](./claude-code-config.example.json) into `~/.claude/settings.json`. Replace the placeholder path with the absolute path to `dist/index.js` in your checkout, and fill in `BRAIN_API_KEY`.

Restart Claude Code. The four tools should appear in the agent's tool list.

## How the agent uses it

- **`brain_query`** — fires at session start to surface relevant decisions and open drift alerts before touching any code.

- **`brain_log_decision`** — fires when a significant architectural choice is made, a library is picked, an approach is rejected, or an unresolved question is flagged. Ingested through the same pipeline as human signals — the next session retrieves it with a citation.

- **`brain_analyze_impact`** — fires before any change that could invalidate a prior decision. Returns which existing decisions the proposed change affects.

- **`brain_log_signal`** — fires when something unexpected is found: a library limitation, API constraint, or behavior that contradicts an existing decision.

The loop closes when Session N+1 retrieves what Session N logged — with no human in the middle.

## Verify it works

From `apps/api`, with the full stack running and `BRAIN_API_KEY` set:

```bash
npm run demo:agent-memory
```

This runs an end-to-end demo: logs a decision via `/brain/agent-log`, waits for the pipeline, then queries as a fresh session and checks that the answer references the logged decision with a citation. PASS means the agent memory loop is closing.
